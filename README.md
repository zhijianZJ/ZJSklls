# ZJSkills

[English](README.en.md)

**版本 [3.0.0](VERSION) · MIT**

把你真实的 AI 职业或学习处境告诉 ZJSkills。它会诊断当前问题，说明证据边界，并给出一个有用的下一步。

ZJSkills 3.0 是一个轻量、AI 优先的职业诊断 Skill。它不会默认发问卷、生成长期课表或推荐课程；它先读取对话中已有的目标、经验、约束和反馈，证据足够就直接行动，只有一个缺失事实会改变判断时，才问**零个或一个**决定性问题。

## 职业诊断

适合“不知道往哪里走”“两个方向怎么选”“我是否适合”“现在是否值得投入”等问题。诊断会区分已知事实、有证据的推断和不确定性，并给出一个最小体验任务，而不是把兴趣、学历、证书或自信直接当作能力证明。

目前重点支持五个可见方向：

- AI Agent 开发
- Vibe Coding / AI 应用开发
- AI 产品经理
- AI 运营（内容/增长、业务效率两个分支）
- AI 工具与职场应用

## 学习路线

方向足够清楚或你明确要求路线时，ZJSkills 会给出不超过三个阶段的紧凑路线。每个阶段先写能力结果、可观察交付物和证据项目，再谈可替换的资源；最后只安排本周最有用的一个行动，并指出最可能改变路线的假设或约束。

默认输出在对话中，**默认不会创建文件**。只有你明确要求保存、导出或持续维护路线时，才创建一份 Markdown 文件，不附带额外状态文件。

## 学习解题

适合概念混淆、懂但不会做、项目不知道如何开始、工具报错、本周没完成、目标改变，或需要消化你提供的材料。它会定位最小卡点，只给一个当前行动、一个可观察的完成信号和一个失败后的检查项，再判断原路线是否需要调整。

## 快速开始

在支持 Skill 的平台输入 `$zjskills` 或 `/zjskills`，也可以直接说真实情况：

```text
我想进入 AI 行业，但不知道 Agent、Vibe Coding、AI 产品还是 AI 运营更适合我。
我目前做运营，会用 AI 写内容，但没有编程项目，每周能投入 6 小时。
请先诊断，不要直接给长期课表。
```

方向明确后可以说“继续，把它展开成不超过三个阶段的学习路线”；学习中可以直接说“我卡住了”并附上现象、报错或材料。

完整复制提示词和逐步示例见[新手入门](docs/getting-started.md)，详细规则见[完整使用手册](docs/usage-guide.md)。

## 非 AI 与安全边界

如果目标主要不属于 AI，ZJSkills 会明确说明，不强行套上 AI 标签。它可以帮助澄清目标、约束、未知项和可迁移的下一步，但在缺少可靠来源时不会冒充该行业专家，也不会臆造岗位标准、市场事实、执照要求或安全规则。

它不替代医疗、法律、财务或心理健康专业判断，不承诺 Offer、转岗、晋升、收入、客户结果或其他由外部主体决定的结果。自学、免费资源、付费课程和结构化支持只按约束、反馈需求、实践机会与证据价值中立比较。

## 安装与平台

技术名称保持为 `zjskills`，仓库地址为：

```bash
git clone https://github.com/zhijianZJ/ZJSkills.git
```

ZJSkills 支持 Codex 和 Claude Code 的原生 Skill 安装；在 Tencent WorkBuddy、豆包以及通用宿主中，当前这个六文件包按手动文件/上下文使用。请按[多平台安装与使用指南](docs/platform-installation.md)完成接入与验收。
