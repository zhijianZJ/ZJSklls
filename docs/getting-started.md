# ZJSkills 新手入门

[English](getting-started.en.md)

你不需要先理解职业模型，也不需要整理一份完整档案。把真实、甚至有些混乱的处境说出来即可。ZJSkills 会先读已有对话；信息足够时直接给诊断，只有一个事实会改变判断时才问一个问题。

## 第一次：直接说真实情况

下面是一段完全可用的第一次输入：

```text
我想进入 AI 行业，但不知道 Agent、Vibe Coding、AI 产品还是 AI 运营更适合我。
我现在做传统运营，平时用 AI 写内容和整理表格，没写过代码，也没有完整项目。
我每周大概能拿出 6 小时，预算不高，希望半年后能拿作品去争取新机会，但我也不确定是否现实。
请先诊断，不要直接给我一堆课。
```

它包含想要的结果、当前工作、可观察经验、时间和预算约束。缺什么不用补全成问卷；如果缺失信息不会改变当前下一步，ZJSkills 会把它列为不确定性并继续。

如果只想复制一句，可以用：

```text
我想进入 AI 行业，但不知道 Agent、Vibe Coding、AI 产品还是 AI 运营更适合我。
请根据我在对话里已经说过的经历先做职业诊断；只有一个缺失事实会改变判断时，才问我一个问题。
```

## 一页职业诊断会包含什么

默认诊断是一页以内的七部分：

1. 你当前的情况；
2. 真正要解决的问题；
3. 当前判断；
4. 判断依据，并分开已知事实、推断和不确定性；
5. 现在先不要做什么；
6. 一个最小验证行动；
7. 如何解释验证结果。

“更适合”“值得测试但有风险”“暂不宜重投入”“证据不足”都是阶段性判断，不是能力证书，也不保证录用、转岗或收入。兴趣、学历、上过课和自我感觉可以作为背景，但不能单独证明能力。

## 回复“继续”：展开学习路线

当方向已经清楚，回复“继续”即可。为了让要求更明确，也可以复制：

```text
方向已经明确，请把它展开成不超过三个阶段的学习路线。
每个阶段先写要形成的能力、可观察交付物和证据项目，再说明资源的用途。
最后只给我本周一个行动和完成标志，并指出最可能改变路线的假设或约束。
```

路线不会自动展开成每天的任务清单。完成一个真实项目或在变化条件下仍能完成目标行为，比“看完课程”更能支持能力判断。

## 说“我卡住了”：进入学习解题

学习中不必重新提交背景，也不必先判断问题类型。直接说：

```text
我卡住了：[粘贴你看到的现象、报错、做过的尝试，或写“不知道从哪开始”]。
请先定位最小卡点，只给我现在做的一个行动、可观察的成功信号和一个失败后的检查项，再说明是否影响原路线。
```

处理概念问题时，它会找到你无法解释或应用的区别；处理报错时，会优先使用已有的完整错误、复现步骤、输入输出和环境；处理没完成的一周时，会先区分偶发打断和持续容量不匹配。它不会因为一次失败就重写整条路线。

## 保存一份 Markdown 路线

默认结果只在对话中。明确要求后，ZJSkills 才会创建一个 Markdown 文件：

```text
请把当前诊断和学习路线保存为一份 Markdown 文件。
文件只包含当前诊断、目标、最多三个阶段、当前行动、证据和简短更新日志；不要创建配套状态文件。
```

以后把新的观察结果告诉它，并要求追加到更新日志。文件路径、写入权限或目标位置不明确时，它应先说明而不是声称已经保存。

## 下一轮怎么说

- 方向仍不确定：“给我一个能区分 Agent 和 Vibe Coding 的最小体验任务。”
- 已完成体验：“我做完了，结果是……请重新判断，不要重复提问。”
- 条件变了：“我每周只能投入 3 小时，请判断只改当前任务、阶段，还是需要重看方向。”
- 有一份材料：“只提取与我当前目标有关的内容，并把它变成一个检索、练习或迁移动作。”
