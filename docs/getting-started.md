# ZJSkills 新手入门

ZJSkills 不是课程推荐器，而是一套结果导向的学习系统设计 Skill。它会先理解你的目标、基础、时间与约束，再把目标反向拆成能力、项目、路线、每周行动和可验证证据，并在条件变化或测评失败时重新规划。

## 使用前准备

不需要先整理完整档案。第一次对话只要尽量提供以下信息：

- 想实现的结果，以及希望在什么时间看到什么证据；
- 当前工作、专业背景和相关经验；
- 每周稳定可投入的时间、预算和设备；
- 英语、数学、编程或业务基础；
- 家庭、工作、健康、隐私等硬约束；
- 最希望形成的真实产物，例如作品集、可部署项目、岗位任务样本或业务交付。

如果信息不确定，可以直接写“未知”或给出范围。自我评价会被视为自述，不会自动当成已经验证的能力。

## 第一次使用

在支持 Skills 的 AI 工具中启用 `learning-architect`，然后复制下面的提示词并替换方括号内容：

```text
请使用 ZJSkills 帮我设计个性化学习系统。
我的目标是：[目标]
当前情况：[工作/专业/相关经验]
每周可投入：[小时]
期望时间：[日期或周期]
主要约束：[预算、英语、设备、家庭、工作等]
希望最终证明自己的方式：[项目、作品集、真实交付、面试任务等]
请先判断还缺少哪些会改变路线的关键信息，每次只问最重要的问题，不要直接推荐课程。
```

第一次回复通常不会立刻给出一张看似完整的课表。Skill 会先明确目标是否可验证、约束是否冲突，以及哪些判断只有假设、哪些已有证据。

## 为什么先提问

同一个“学习 AI”目标可能对应完全不同的系统：了解行业、在原岗位提效、转岗 AI 产品经理、转岗 AI Agent 工程师或完成企业项目，需要的能力证据和项目都不同。时间、预算或基础变化，也会改变路线顺序。

Skill 只追问会实质改变决策的信息。你不必一次回答发现问卷中的全部问题，可以：

1. 先回答目标、基线、每周容量和硬约束；
2. 对未知项明确写“未知”；
3. 允许低风险部分以“草案 + 假设 + 验证动作”继续；
4. 对求职、合规、医疗、财务、真实生产部署等高风险结论补充可靠来源或人工复核。

## 如何继续

关键问题澄清后，可以发起第二轮请求：

```text
基于已经确认的信息，生成第一版完整学习系统：目标结果、能力差距、能力树、知识依赖图、项目阶梯、阶段路线、未来一周计划、评估标准和下一次复盘条件。请区分事实、自述、证据、推断和假设，并指出当前未通过的闸门。
```

后续不必重新开始。若希望跨任务保存，请先指定一个你授权的学习者工作区，并要求写入 `system-state.yaml`；Skill 不会自行选择全局保存位置，没有写文件权限时也不会声称已经持久化。常用继续方式：

```text
读取并验证[学习者工作区路径]/system-state.yaml，继续上次的 ZJSkills 状态，从最早未通过的阶段开始。
```

```text
这是本周的新证据：[链接、文件、测试结果、反馈或复盘]。请更新能力判断，只调整被证据影响的部分。
```

```text
我本周实际只有 4 小时。请保留目标，重新计算容量并给出最低可交付版本。
```

每轮结果应同时包含简短的人类可读说明与 `engine_result` 结构化状态。只有实际写入文件的产物才应出现在 `artifacts_written` 中。

## 如何重新规划

重新规划不是在旧计划后面简单加任务，而是按触发原因回滚到最早受影响的决策：

- 目标改变：回到 Goal Analysis，并重算所有下游产物；
- 时间、预算或环境改变：重查 Roadmap 与 Weekly Planner；
- 项目或测评失败：定位最早的因果能力差距，增加针对性练习、调试或复盘；
- 行业或技术资料更新：重查 Domain Pack 及能力、课程和项目设计；
- 证据互相矛盾：保留冲突、降低置信度，安排最小区分性测评。

可直接使用：

```text
我的目标从[原目标]变为[新目标]，原因是[原因]。请创建新版本，列出受影响的下游产物，并从正确阶段重新规划，不要覆盖历史版本。
```

```text
本阶段没有达到评分阈值，失败证据是[内容]。请找出最早因果缺口，设计从指导完成到独立完成的补强方案，然后更新后续路线。
```

ZJSkills 提升的是做出学习决策、积累能力证据和持续调整的质量；它不保证 Offer、转岗、涨薪、收入、创业成功或任何由外部主体决定的结果。
