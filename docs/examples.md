# ZJSkills 使用场景与提示词

[English](examples.en.md)

这些示例展示 3.0 的真实入口。提示词可直接复制；方括号替换为你的观察结果。理想回复应保持紧凑、复用已有上下文、只选一种模式，并以一个行动结束。

## 场景一：模糊的 AI 转型

```text
我做了 5 年行政，想转 AI，但只会日常使用聊天工具，不知道学什么。每周大约 5 小时，也不确定要不要辞职学习。请先诊断。
```

应进入职业诊断，区分“想进入 AI”的愿望与尚未明确的工作对象。若现有事实足够，给一个最小体验任务；若目标结果是唯一决定性缺口，只问这一个问题。不要先给半年课表，也不要建议辞职。

## 场景二：Agent 与 Vibe Coding

```text
我会一点 Python，跟着教程做过聊天机器人，但没独立改过需求。我在 AI Agent 开发和 Vibe Coding 之间选不定。请给我一个最小任务来区分，不要凭兴趣下结论。
```

应说明教程跑通不是独立能力证据。区分任务可以要求先构建一个小工具并验证需求变化，再加入模型/API 调用和一个可诊断故障；根据理解、修改、测试、调试的结果重新判断。

## 场景三：没有编程证据

```text
我没有写过代码，但很想做 AI Agent 工程师。我每天刷很多 Agent 视频，感觉自己理解得不错。你觉得适合吗？
```

应把愿望和视频学习列为已知背景，但标明没有编程或项目证据。合理判断是“证据不足先验证”或“暂不宜重投入”，下一步只给一个边界清楚的模型/API 小任务及成功、失败的解释方式。

## 场景四：自学还是结构化支持

```text
我每周有 8 小时，预算有限，已经自学两个月，但遇到报错常常停一周。我能按明确任务完成练习，却很难自己判断项目是否达标。我是否必须报班？
```

应中立比较：自学成本较低且已有执行证据；结构化支持可能提供及时反馈和验收，但价格或品牌不能证明效果。可以先设计一次有明确评分标准和外部反馈的短实验，再决定是否需要持续支持，而不是导向某个提供者。

## 场景五：概念混淆

```text
我分不清提示词里的“上下文”和“指令”。文章都看过，但换一个例子就不会判断。我卡住了。
```

应进入学习解题，指出卡点是无法在新例子中应用区别。只给一个动作，例如标注一段真实提示词中“提供判断材料”的上下文和“要求模型做什么”的指令；成功信号是能解释两个标注并处理一个变化示例。

## 场景六：项目报错

```text
我的 Python 项目调用模型时返回 401。我已经重装了依赖，还是不行。环境是 macOS，完整错误是：[粘贴错误]。我卡住了。
```

应先使用已有环境、错误和尝试，不重复让用户“描述问题”。第一行动应是安全地核对认证配置是否被当前进程读取，成功信号必须可观察；不得要求用户粘贴密钥，也不得在证据不足时一次罗列所有可能原因。

## 场景七：一周没完成

```text
这周临时加班，我计划的 6 小时只做了 1 小时，项目没交付。下周时间会恢复。我是不是应该把整条路线推迟一个月？
```

应把它视为一次已说明的临时中断，不自动重做阶段路线。只调整当前任务或下周的最小交付，并给一个完成标志；如果后续反复发生，再用新证据判断是否存在持续容量失配。

## 场景八：目标变了

```text
我原来准备转 AI Agent 开发，但公司给了我一个 AI 产品内部转岗机会。我更想争取这个机会，三个月后要做方案评审。请判断原路线哪些还能用。
```

应识别为目标变化，重新做职业诊断或在证据足够时生成 AI 产品路线。保留可迁移的模型理解和项目经验，只重构受影响阶段；不要把原路线整体删除，也不要把“内部机会”当成录用保证。

## 场景九：非 AI 请求

```text
我想在半年内转做注册营养师。请给我权威考试要求、就业行情和完整学习路线。
```

应明确该目标主要不属于 AI，而且现有 Skill 没有建立当地执照、考试和就业市场依据。它可以先确认地区和用户已有的官方材料，帮助整理目标与验证下一步，但不能冒充营养或执照专家，更不能编造权威标准。

更多输出结构见[完整使用手册](usage-guide.md)，第一次使用见[新手入门](getting-started.md)。
