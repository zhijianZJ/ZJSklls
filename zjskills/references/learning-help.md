# Learning Help

Use this mode for concept confusion, an action gap, difficulty starting a project, tool or error evidence, missed work, a changed goal, or material the user supplies.

First locate the smallest current blocker:

- **Concept confusion:** identify the distinction the user cannot yet explain or apply.
- **Action gap:** convert passive understanding into one observable attempt.
- **Project start:** shrink scope to the first testable slice or deliverable.
- **Tool or error evidence:** use the exact message, reproduction step, input, output, and environment already available; ask for only the decisive missing item.
- **Missed work:** distinguish a one-off interruption from a persistent capacity mismatch.
- **Changed goal:** identify whether the target or only the current task changed.
- **Supplied material:** classify relevant claims as fact, opinion, example, inference, or unknown, then choose one retrieval, practice, or transfer action.

Return the following headings in this exact order. Give one action, not a list of parallel tasks.

## Where you are stuck

Name the precise point where progress stopped using observed evidence.

## Most likely cause

State the leading cause and label uncertainty when evidence is incomplete.

## Do this one action first

Give one bounded action the user can perform now. For supplied material, make it one retrieval, practice, or transfer action.

## Observable success signal

Describe what the user can see, run, explain, compare, or deliver if the action works.

## If it fails, check this next

Name one fallback check that distinguishes the next most likely cause.

## Route impact

Say whether the existing route is unchanged, needs a current-task adjustment, needs a stage adjustment, or must be reconsidered because the goal changed. Do not rewrite the route without evidence.
